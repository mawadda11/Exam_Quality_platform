# Review Code

Review the current changes before they are committed.

1. Read `CLAUDE.md` and the relevant project documents.
2. Inspect `git diff` and identify correctness, security, privacy, traceability, and test issues.
3. Verify that no secrets, real exam files, student data, or generated private reports are included.
4. Verify that every finding remains evidence-based and uses exactly one allowed academic status.
5. Run the relevant backend and frontend tests and linters.
6. Report blockers first, then recommended improvements.
7. Do not modify files until a plan is shown and approved when the requested change is substantial.
